# PRD 生成技能

帮助用户创建结构化的产品需求文档，用于 Ralph 自主代理执行。

## 工作流程

1. 接收功能描述
2. 询问澄清问题（3-5个）
3. 生成结构化 PRD
4. 保存到 `tasks/prd-[feature].md`

## PRD 结构

生成的 PRD 包含:

1. **概述** - 功能简介
2. **目标** - 可衡量的目标
3. **用户故事** - 详细的用户故事
4. **技术方案** - 实现建议
5. **非功能需求** - 性能、安全等
6. **风险与缓解** - 潜在风险

## 用户故事模板

每个用户故事包含:
- ID (US-001, US-002, ...)
- 标题
- 描述 (As a... I want... so that...)
- 验收标准 (可检查列表)
- 优先级 (1-5)
- 状态 (待办)

## 示例

用户: "创建一个用户认证系统"

AI: "为了创建准确的 PRD，我需要了解一些信息:

1. 认证方式?
   A. 邮箱+密码
   B. 手机号+验证码
   C. OAuth (Google/GitHub)
   D. 多种方式

2. 需要哪些功能?
   A. 仅登录/注册
   B. + 密码重置
   C. + 邮箱验证
   D. + 双因素认证

..."

然后生成完整的 PRD 文档。
